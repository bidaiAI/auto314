# Autonomous 314 Trademark and Brand Notice

Last updated: **2026-04-11**

Source code in this repository is published under the [MIT License](./LICENSE), but that does **not** grant permission to present a fork as the official Autonomous 314 product.

## Reserved names and identifiers

The following names and official identifiers are used by the official deployment and should be treated as protected brand identifiers:

- `Autonomous 314`
- `auto314`
- `auto314.cc`
- `@Auto314_CC`
- `@Autonomous314`
- `@auto314_Alert`

## Brand assets that forks should replace

If you deploy a fork publicly, replace the official brand assets before launch, including the current assets under:

- `apps/web/public/brand/`
- `apps/web/public/favicon*`

## What forks may do

Forks may:

- use the code under MIT,
- study, modify, and self-host the protocol,
- describe themselves factually as being derived from this repository.

## What forks should not do without permission

Forks should **not**:

- imply they are the official Autonomous 314 deployment,
- reuse the official project identity in a way that confuses users,
- reuse official logos, favicon, or channel identities for a public deployment without permission,
- present official domains or official social channels as if they belong to the fork.

## Recommended fork practice

Before deploying a fork publicly:

1. replace name, logo, favicon, and channel links,
2. replace deployment env values and callback URLs,
3. point frontend and indexer configs at your own infrastructure,
4. clearly state that the fork is unofficial.
